## 1. 系统概述

### 1.1 项目背景

高校数字化建设积累了多源异构的学生行为数据（成绩、消费、借阅、上网、门禁等），但缺乏直观、智能的分析工具。本项目构建一个轻量级学业行为分析系统，帮助辅导员快速识别学生的优势与风险，并为学生提供个性化学习建议。

### 1.2 设计目标

- 构建清晰的三层架构，前后端分离，模块间低耦合高内聚
- 支持 Docker Compose 一键部署
- 五维加权风险评分模型替代简单挂科数判定
- 集成大模型 API，提供个性化学习建议（含降级兜底）
- 班级群体分析看板支持多维度交叉分析

### 2.2 关键架构决策

| 决策点 | 选择 | 理由 |
| :--- | :--- | :--- |
| 前后端分离 | Vue 3 独立项目 + Spring Boot REST API | 解耦开发，nginx 统一入口 |
| ORM | Mybatis-Plus | 简化单表 CRUD，复杂查询用 XML |
| 认证方案 | JWT (无状态) | 轻量，无需服务端 session |
| 标签计算 | 查询时动态计算 | 避免数据冗余，保证标签始终基于最新数据 |
| 风险评分 | Service 层聚合计算，结果写入 risk_log | 历史可追溯，查询不重复计算 |
| AI 调用 | 同步请求 + 5s 超时 + 降级兜底 | 保证用户体验，避免阻塞 |

## 3. 模块划分

### 3.1 模块总览

系统划分为 6 个功能模块，各模块在后端对应独立的 Service，在前端对应独立的路由页面或组件。

| 模块 | 后端 Service | 前端路由/组件 | 职责 |
| :--- | :--- | :--- | :--- |
| 数据管理 | DataImportService | `/data-import` | CSV 导入、预览、清空 |
| 用户认证 | AuthService | `/login` + 路由守卫 | 登录认证、权限控制 |
| 学生画像 | StudentService | `/students` + `/students/:id` | 学生列表、详情画像 |
| 风险评分 | RiskScoreService + BehaviorTagService | 嵌入学生列表/详情 | 五维评分、标签生成 |
| AI 建议 | AiSuggestionService | 嵌入学生详情页 | LLM 调用、降级兜底 |
| 班级看板 | DashboardService | `/dashboard` | 5 区群体分析看板 |

### 3.2 模块详细职责

#### 3.2.1 数据管理模块

- **职责**：接收 6 类 CSV 文件上传，解析并批量写入对应数据库表
- **输入**：6 类 CSV 文件（学生基本信息、成绩、消费、借阅、上网行为、门禁记录）
- **输出**：写入 student / course_score / consumption / library_borrow / internet_behavior / dormitory_access 表
- **子功能**：
  - CSV 解析与校验（必填字段检查、格式校验）
  - 批量入库（Mybatis-Plus batch insert）
  - 最近导入数据预览（最近 N 条）
  - 一键清空业务表（保留 user 表）

#### 3.2.2 用户认证模块

- **职责**：用户登录认证、JWT 签发与校验、前端路由守卫、角色权限控制
- **输入**：用户名 + 密码
- **输出**：JWT token（含 role、student_id 等 claims）
- **子功能**：
  - BCrypt 密码加密存储与校验
  - JWT 生成与过期管理（≥ 1 天）
  - 前端路由守卫（未登录重定向到 `/login`）
  - 角色权限：学生只能看本人数据，辅导员可看本班数据

#### 3.2.3 学生画像模块

- **职责**：展示学生列表（增强）和学生综合画像详情
- **输入**：学生 ID、班级筛选条件
- **输出**：学生列表视图 / 学生详情视图
- **子功能**：
  - 增强列表：姓名、学号、班级、风险等级标签、行为标签、五维迷你进度条
  - 多条件筛选：按风险等级、行为标签、风险维度
  - 模糊搜索：姓名 / 学号
  - 详情页：五维雷达图（叠加班级均值虚线）、行为标签条、风险明细、各维度趋势折叠面板

#### 3.2.4 风险评分模块

- **职责**：五维加权评分计算、行为标签动态生成、风险等级判定
- **输入**：学生各维度聚合数据
- **输出**：五维得分（各 0-20 分）、综合总分（0-100）、风险等级（高/中/低）、行为标签列表
- **子功能**：
  - 五维评分：学业水平 / 借阅投入 / 消费健康 / 上网健康 / 作息规律
  - 规则标签生成（7 个绝对值标签）
  - 相对标签生成（5 个班级排名标签）
  - 风险记录持久化（risk_log 表，含各维度明细 JSON）
  - 筛选接口（按风险等级/标签/维度得分筛选）

#### 3.2.5 AI 建议模块

- **职责**：构造全维度 Prompt，调用大模型 API，返回个性化学习建议
- **输入**：学生全部五维数据 + 风险等级 + 行为标签
- **输出**：个性化建议文本（含优劣势分析），或降级兜底文案
- **子功能**：
  - Prompt 构造（全维度模板填充）
  - HTTP 调用大模型 API（5 秒超时）
  - 降级处理：超时/网络错误 → 兜底文案
  - 调用限制：每学生每天最多 3 次

#### 3.2.6 可视化看板模块

- **职责**：班级群体分析，5 区联动展示
- **输入**：班级 ID、学期筛选
- **输出**：5 区图表数据
- **子功能**：
  - ① 风险概览：饼图 + 趋势折线
  - ② 五维均值：班级 vs 年级叠加雷达图
  - ③ 异常关注列表：各维度 Top N 异常学生
  - ④ 共性分析：高风险群体 vs 全班均值差异文本
  - ⑤ 交叉分析：4 张散点图（各维度 vs GPA）

### 4.3 关键依赖说明

| 依赖关系 | 说明 |
| :--- | :--- |
| 数据管理 → 所有业务模块 | 数据管理是基础模块，所有其他模块都依赖它导入的数据库表 |
| 风险评分 → 学生画像 | 学生列表和详情页展示风险等级、行为标签，均来自风险评分计算 |
| 风险评分 → AI 建议 | AI Prompt 中包含风险等级和行为标签作为输入上下文 |
| 风险评分 → 班级看板 | 看板的风险概览区、异常关注列表直接读取风险评分结果 |
| 学生画像 → AI 建议 | AI 建议按钮嵌入学生详情页，由画像模块触发 |
| 用户认证 → 所有业务模块 | 横切关注点，所有业务接口都需要 JWT 校验 |

**模块独立性说明**：
- **数据管理**和**用户认证**是两个独立的基础模块，互不依赖
- **班级看板**和**学生画像**彼此独立，可并行开发
- **AI 建议**虽然在 UI 上嵌入画像页，但后端 Service 独立，可单独测试

### 5.2 核心数据流

```
  CSV 文件                         数据库表                      前端展示
 ──────────                      ──────────                    ──────────

 学生信息.csv  ──┐                                       ┌→ 学生列表
                 │    ┌─────────────┐                    │
 成绩.csv  ──────┤    │             │    student         │→ 雷达图
                 ├───▶│ DataImport  │    course_score    │

                 │    │             │    consumption     │
 消费.csv  ──────┤    └──────┬──────┘    internet_behavior│→ AI Prompt
                 │           │           dormitory_access │
 上网.csv  ──────┤           ▼                           │→ 看板图表
                 │    ┌─────────────┐    risk_log ───────┘
 门禁.csv  ──────┘    │ RiskScore   │
                      │ Service     │
                      │ (聚合计算)   │
                      └─────────────┘
```

**数据流步骤**：

1. **CSV 导入**：DataImportService 解析 CSV → 逐行校验 → 批量写入 6 张业务表
2. **风险计算**：RiskScoreService 按 student_id 聚合各维数据 → 五维评分 → 写入 risk_log
3. **画像查询**：StudentService 联表查询 student + 聚合业务数据 + risk_log → 返回前端
4. **AI 建议**：AiSuggestionService 读取全部维度数据 → 构造 Prompt → 调用 LLM → 返回文本
5. **看板查询**：DashboardService 按班级聚合 → 计算均值/分布/排名 → 返回图表数据

## 7. 技术选型说明

| 技术 | 版本 | 选型理由 |
| :--- | :--- | :--- |
| Java | 17 | LTS 版本，Spring Boot 3.x 最低要求 |
| Spring Boot | 3.x | 课程要求；自动配置、生态成熟；AOT 支持 |
| Mybatis-Plus | 3.5+ | 简化单表 CRUD，减少 XML 编写；分页插件；逻辑删除支持 |
| Maven | 3.8+ | 课程要求；依赖管理、多模块构建 |
| MySQL | 8.0 | 课程要求；窗口函数支持（班级排名计算）；JSON 类型（dimension_scores） |
| Vue 3 | 3.x | 课程要求；组合式 API；响应式数据绑定 |
| Tailwind CSS | 3.x | 快速 UI 开发，无需独立 CSS 文件，与 Vue 3 生态兼容 |
| ECharts | 5.x | 雷达图、散点图、饼图、折线图一站支持；Canvas 渲染性能好 |

## 8. 关键设计决策

### 8.1 标签动态计算 vs 静态存储

**决策**：行为标签在查询时动态计算，不存储固定值。

**理由**：
- 标签依赖班级排名（相对标签），班级数据变化时排名自动更新
- 避免数据不一致（修改成绩后标签仍显示旧状态）
- 标签计算逻辑集中在 BehaviorTagService，单一职责，便于调整规则

**风险缓解**：列表查询时需批量计算标签，可能影响性能。措施：
- 标签计算所需数据一次性批量查询（避免 N+1）
- 班级排名数据缓存到请求级变量

### 8.2 风险评分计算时机

**决策**：数据导入后触发批量重算，结果持久化到 risk_log 表。

**理由**：
- 风险评分涉及多表聚合，查询时实时计算会拖慢列表接口
- risk_log 保留历史记录，可追溯风险变化趋势
- 看板的"近 3 学期风险趋势"依赖历史 risk_log 数据

**触发方式**：辅导员在数据管理页导入 CSV 后，前端调用 `/api/risk/recalculate` 接口。

### 8.3 AI 降级策略

**决策**：同步调用 + 5 秒超时 + 兜底文案。

```
调用大模型 API
    │
    ├─ 成功 (HTTP 200) ──▶ 返回建议文本
    │
    ├─ 超时 (5s)    ──▶ 返回兜底文案 + 提示"服务繁忙"
    │
    ├─ 网络错误      ──▶ 返回兜底文案 + 提示"网络异常"
    │
    └─ 非 200 状态码 ──▶ 返回兜底文案 + 提示"服务暂不可用"
```

**理由**：
- 5 秒是用户体验容忍上限
- AI 不是核心路径，降级不阻塞主流程
- 兜底文案保留基本价值，不是冷冰冰的错误提示

### 8.4 前后端分离的 API 设计原则

- RESTful 风格，URL 使用资源名词（`/api/students`、`/api/dashboard`）
- 统一响应格式：`{ code, message, data }`
- 后端全局异常处理（`@RestControllerAdvice`）+ 参数校验（`@Valid`）
- 前端 Axios 拦截器统一附加 JWT、处理 401 跳转登录

### 8.5 安全设计

| 措施 | 实现 |
| :--- | :--- |
| 密码存储 | BCrypt 加密，不可逆 |
| 认证 | JWT，过期 ≥ 1 天，前端存储于 localStorage |
| 权限控制 | 学生：只能查询本人 student_id 的数据；辅导员：只能查询本班 class_name 的数据 |
| CSRF | 本地开发环境下忽略 |
| SQL 注入 | Mybatis-Plus 参数化查询，无拼接 SQL |
