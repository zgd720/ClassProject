## 1. 引言

### 1.1 编写目的
本文档旨在定义"基于多源数据的大学生学业行为分析助手"的功能需求、非功能需求、数据需求及技术约束，作为项目设计、开发、测试及验收的依据。

### 1.2 项目背景
高校数字化建设积累了多源异构的学生行为数据（成绩、消费、借阅、上网等），但缺乏直观、智能的分析工具。本项目通过构建轻量级学业行为分析系统，帮助辅导员快速了解学生学业状态，并为学生提供个性化学习建议。

### 1.3 项目目标
- 构建学生基础画像（学业、消费、借阅等多维度）。
- 实现简单规则驱动的学业风险提示（低/中/高）。
- 集成大模型API，根据学生数据生成个性化学习建议。
- 提供班级数据可视化看板。
- 支持Docker Compose一键部署。

### 1.4 用户角色
| 角色 | 说明 | 主要操作 |
| :--- | :--- | :--- |
| 辅导员 | 查看所带班级学生数据 | 导入数据、查看学生画像、生成AI建议、查看看板 |
| 学生 | 查看个人画像和建议 | 登录、查看本人信息及建议 |
| 管理员 | 系统初始化与用户管理（可选） | 用户导入、角色分配 |

## 2. 总体描述

### 2.1 核心业务流程
1. 辅导员通过CSV文件导入学生信息、成绩、消费、借阅记录。
2. 系统清洗并存储数据，自动计算每个学生的风险等级（基于挂科数、绩点）。
3. 辅导员查看学生列表及风险标签，点击学生进入详情页查看画像。
4. 点击"生成AI建议"，系统调用大模型API并返回个性化学习建议。
5. 辅导员或学生可查看班级/个人的可视化统计看板。

### 2.2 假定与依赖
- 外部大模型API可用（免费或低配额测试Key）。
- 开发环境支持Docker。
- 数据集采用赛题样本数据或自行生成的模拟数据。

## 3. 功能需求

### 3.1 模块结构

| 模块 | 子功能 | 对应角色 |
| :--- | :--- | :--- |
| 数据管理模块 | CSV导入、数据预览、数据清空 | 辅导员/管理员 |
| 学生画像模块 | 学生列表（搜索、分页）、详情页（学业/消费/借阅趋势） | 辅导员、学生 |
| 风险提示模块 | 自动风险等级计算与标记、风险学生筛选 | 辅导员 |
| AI建议模块 | 调用大模型生成个性化建议（降级兜底） | 辅导员、学生 |
| 可视化看板模块 | 班级风险分布、绩点趋势、消费时段分布、借阅类别统计 | 辅导员 |
| 用户认证模块 | 登录、登出、路由守卫 | 所有用户 |

### 3.2 详细功能描述

#### 3.2.1 数据管理模块
- **CSV导入**：支持上传CSV文件，包含学生基本信息表、成绩表、消费记录表、借阅记录表。后端解析后批量写入对应数据库表。
- **数据预览**：展示最近导入的若干条记录，供用户确认。
- **数据清空**：提供一键清空所有业务表（保留用户表）的功能，用于重新导入。

#### 3.2.2 学生画像模块
- **学生列表**：展示姓名、学号、班级、当前平均绩点、挂科数、风险等级（颜色标签）。支持按姓名/学号模糊搜索。
- **学生详情页**：
  - 基本信息：姓名、学号、专业、班级、入学年份。
  - 学业趋势：各学期平均绩点折线图。
  - 借阅统计：累计借阅量，近30天借阅频次。
  - 消费统计：日均消费金额，消费类型占比（餐饮/超市等）。
  - 风险等级及判定理由（例如：挂科数≥2 → 高风险）。

#### 3.2.3 风险提示模块
- **风险规则**：
  - 高风险：挂科数 ≥ 2 或 平均绩点 < 2.0。
  - 中风险：挂科数 = 1。
  - 低风险：其他。
- **功能**：列表页支持按风险等级筛选；详情页展示风险等级及具体原因。

#### 3.2.4 AI建议模块（强制）
- **触发方式**：在学生详情页提供"生成AI建议"按钮。
- **后端调用**：Spring Boot Controller接收请求，收集该学生的绩点、挂科数、借阅量、日均消费等数据，构造Prompt，调用外部大模型API（HTTPS），获取响应文本并返回前端。
- **Prompt设计示例**：
  > 你是一名大学辅导员，请根据以下学生数据给出学习建议：
  > 平均绩点：{gpa}，挂科数：{failCount}，本学期借阅次数：{borrowCount}，日均消费金额：{avgConsume}元。
  > 要求：建议具体、鼓励性强，不超过200字。
- **降级处理**：若API调用超时（5秒）、网络错误或返回非200状态码，则返回固定兜底文案："当前无法获取AI建议，请稍后再试。建议您多与任课老师沟通，并合理安排学习时间。"
- **记录要求**：在 `docs/ai_module.md` 中记录模型名称、API提供方、请求/响应示例、Prompt截图、限流/成本控制说明。

#### 3.2.5 可视化看板模块
- **班级风险分布**：饼图展示高/中/低风险学生占比。
- **绩点趋势**：柱状图展示全班各学期平均绩点的变化。
- **借阅类别分布**：饼图展示借阅图书类别（文学、科技、社科等）占比。
- **消费时段分布**：折线图展示一天内各时段的消费金额总和。

#### 3.2.6 用户认证模块
- 使用JWT实现登录认证。
- 前端路由守卫：未登录状态下无法访问除登录页外的任何页面。
- 简化权限：学生登录后只能查看本人数据；辅导员可查看本班所有学生数据（班级字段关联）。

### 3.3 非功能需求

| 类别 | 具体要求 |
| :--- | :--- |
| 性能 | 单次API请求≤10秒；列表查询响应≤2秒；看板加载≤5秒。 |
| 可用性 | 前端提供Loading状态；所有操作均有成功/失败提示。 |
| 可靠性 | 后端全局异常处理；AI模块降级不导致主流程崩溃。 |
| 可维护性 | 遵循三层架构；Swagger文档可访问；关键代码注释。 |
| 安全性 | 密码使用BCrypt加密；JWT过期时间≥1天；CSRF忽略（本地开发）。 |

## 4. 数据需求

### 4.1 数据表设计（至少5张）

| 表名 | 主要字段 | 说明 |
| :--- | :--- | :--- |
| `user` | id, username, password, role, name, student_id(可为空) | 用户表（辅导员/学生） |
| `student` | id, student_id, name, gender, major, class_name, enrollment_year | 学生基本信息 |
| `course_score` | id, student_id, semester, course_name, score, gpa | 成绩表，gpa为绩点 |
| `library_borrow` | id, student_id, borrow_date, book_name, category | 借阅记录 |
| `consumption` | id, student_id, consume_time, amount, type | 消费记录 |
| `risk_log`（可选） | id, student_id, risk_level, update_time | 记录风险等级变更 |

> 所有表均包含 `create_time`, `update_time`, `deleted` (逻辑删除) 字段。

### 4.2 数据来源
- 使用赛题提供的样本数据或自行生成的模拟数据（CSV格式）。
- 数据量要求：学生数≥20人，每人至少3门课成绩，≥10条借阅记录，≥20条消费记录。

## 7. 验收标准

- 执行 `docker compose up --build` 后，可通过 `http://localhost` 访问前端，通过 `http://localhost:8080/swagger-ui/index.html` 访问API文档。
- 辅导员可成功导入CSV文件，学生列表正确显示风险标签。
- 点击学生详情页可看到画像及风险等级。
- 点击"生成AI建议"按钮，能在合理时间内返回非空洞的建议内容（或降级文案）。
- 可视化看板图表正确展示统计数据。
- GitHub提交记录≥8次，Claude对话记录≥3次，所有文档完整。
